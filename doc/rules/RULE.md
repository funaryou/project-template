# Rules Directory — ルール定義書のフォーマット

> **目的**: `doc/rules/` に配置するルール定義書（`*-rule.md`）の統一フォーマットを定める。
> **対象**: このディレクトリにルールファイルを追加・編集するすべての開発者・AIアシスタント

---

## 1. このディレクトリの役割

`doc/rules/` は、特定カテゴリの文書を書くための **フォーマット定義ルールファイル** を格納する。

| 役割 | 説明 |
|------|------|
| 文書品質の統一 | 同じカテゴリの文書がバラバラな構成にならないよう、テンプレートを提供する |
| 効率的な生成 | AIアシスタントがルールに沿った文書を素早く生成できるようにする |
| レビュー基準の明確化 | 「この構成に沿っているか」をレビューの判断基準にする |

---

## 2. ルールファイルのフォーマット

各ルールファイル（`*-rule.md`）は以下の構成に従うこと。

### 2.1 必須セクション

```markdown
# {カテゴリ名} — {タイトル／補足}

> **目的**: {このルールの目的を一文で}
> **対象**: {このルールが適用される対象}

---

## 1. {基本原則など、最初の大項目}

...

---

## N. フォーマットチェックリスト

### {対象文書カテゴリ}（`{ファイル名パターン}`）
- [ ] チェック項目1
- [ ] チェック項目2
```

### 2.2 セクションの要件

| セクション | 必須 | 説明 |
|-----------|------|------|
| 冒頭の目的・対象 | 必須 | ルールの意図とスコープを明確にする |
| 基本原則 | 任意 | 守るべき根本的な考え方 |
| 構成ルール | 任意 | 必須セクションや順序の定義 |
| コードの扱い | 任意 | コードブロックの許可条件 |
| 解説の書き方 | 任意 | 文体や粒度のルール |
| フォーマットチェックリスト | 推奨 | 作成後に確認するチェック項目 |

---

## 3. ルール間の参照

- ルールファイルは相互にリンクしてよい。
- より汎用的なルール（例: 全文書共通ルール）は `doc/RULE.md` に記述し、`doc/rules/` の各ルールはそれを参照する形にする。
- ルール同士で矛盾が生じた場合は、`doc/RULE.md` → `doc/rules/RULE.md` → 個別ルール の優先順位で解決する。

---

## 4. 新規ルール追加手順

1. この `RULE.md` のフォーマットに従って `{category}-rule.md` を作成する。
2. 必要に応じて `doc/RULE.md` の `## 3. Directory Responsibilities` に追記する。
3. 既存の類似ルールと内容が重複していないか確認する。
