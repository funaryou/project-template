# Document Rules — プロジェクト共通の文書フォーマットルール

このリソースは、本プロジェクトで文書（特にガイド系ドキュメント）を作成する際に参照すべきルールを定義します。

## 重要: 以下のファイルを常に読み込むこと

このプロジェクトで文書を作成・編集する際は、**必ず以下のルールファイルを参照してから作業を開始すること**。

| ファイル | 内容 | 優先度 |
|---------|------|--------|
| `docs/RULE.md` | プロジェクト全体のドキュメント構造・命名・運用ルール | 最優先 |
| `docs/rules/RULE.md` | `docs/rules/` ディレクトリの定義とルールファイルのフォーマット | 高 |
| `docs/rules/doc-guid-rule.md` | ガイド系ドキュメント（`*_guide.md` / `*_answers.md`）の具体的なフォーマットルール | 高（ガイド作成時） |

## ルールの優先順位

1. `docs/RULE.md` — プロジェクト全体の共通ルール（最優先）
2. `docs/rules/RULE.md` — ルール定義書のメタフォーマット
3. `docs/rules/*-rule.md` — カテゴリ別の個別ルール（対象に応じて適用）

## 適用タイミング

- **ガイド系ドキュメントを作成するとき**: `docs/rules/doc-guid-rule.md` の全セクションに従うこと
- **新しいルールを追加するとき**: `docs/rules/RULE.md` のフォーマットに従い、`docs/RULE.md` の Directory Responsibilities も更新すること
- **ふだんのコーディング・設計**: `docs/RULE.md` の基本ルール（命名・配置・.gitignore 等）を遵守すること

## リマインダー

- ガイドには完成コードを書かない（疑似コード・関数シグネチャ・一部抜粋のみ）
- 答案コード集はガイドとセットで作成する
- 新しい文書カテゴリが必要な場合は、先にルールを定義してから文書を作成する
