# API ドキュメント（テンプレート）

このドキュメントでは、ドキュメント連携アプリで利用可能な API エンドポイント例を説明します。

## ベース URL
`http://<host>:<port>/api`

---

## 投稿 (Posts)

通常の投稿（テキスト、リンク、およびファイル）に関するエンドポイントです。

### 1. 投稿一覧の取得
- **URL**: `/posts`
- **メソッド**: `GET`
- **説明**: すべての投稿を作成日時の新しい順に取得します。

### 2. 個別投稿の取得
- **URL**: `/posts/:id`
- **メソッド**: `GET`
- **説明**: 指定した ID の投稿詳細を取得します。

### 3. 新規投稿の作成
- **URL**: `/posts`
- **メソッド**: `POST`
- **Content-Type**: `multipart/form-data`
- **ボディ**:
  - `text` (任意): プレーンテキストの内容
  - `link` (任意): リンク URL 文字列
  - `file` (任意): ファイルアップロード（画像、PDF など）

### 4. 投稿の更新
- **URL**: `/posts/:id`
- **メソッド**: `PUT`
- **Content-Type**: `multipart/form-data`
- **ボディ**: 新規投稿作成と同じ

### 5. 投稿の削除
- **URL**: `/posts/:id`
- **メソッド**: `DELETE`
- **説明**: 投稿を削除し、関連ファイルもストレージから消去します。

## 記事 (Articles)

Markdown 形式の記事に関するエンドポイントです。

### 1. 記事一覧の取得
- **URL**: `/articles`
- **メソッド**: `GET`
- **説明**: すべての記事を作成日時順に取得します。

### 2. 個別記事の取得
- **URL**: `/articles/:id`
- **メソッド**: `GET`
- **説明**: 指定した ID の記事詳細を取得します。

### 3. 新規記事の作成
- **URL**: `/articles`
- **メソッド**: `POST`
- **Content-Type**: `application/json`
- **ボディ**:
  ```json
  { "content": "Markdown の内容..." }
  ```

### 4. 記事の更新
- **URL**: `/articles/:id`
- **メソッド**: `PUT`
- **Content-Type**: `application/json`
- **ボディ**:
  ```json
  { "content": "更新された Markdown の内容..." }
  ```

### 5. 記事の削除
- **URL**: `/articles/:id`
- **メソッド**: `DELETE`
- **説明**: 記事を削除します。

---

## ユーティリティ

### 1. ローカル IP の取得
- **URL**: `/ip`
- **メソッド**: `GET`
- **説明**: サーバーのローカル IP アドレスを返します（QR コード生成用）。
