# Design Principles

このプロジェクトのデザイン系スキルは、以下の原則を共通前提として扱います。

## Core Principles

1. 見た目の派手さより、情報構造の明快さを優先する
2. キャラクター性は残しつつ、読みやすさと操作の迷いにくさを壊さない
3. 色だけで状態を伝えず、ラベル、形状、余白でも意味を補強する
4. PC とモバイルで破綻しないことを最低条件にする
5. デザイン提案は、実装時の分割単位まで意識して行う

## Review Axes

- Visual hierarchy
- Information grouping
- Readability
- Interaction clarity
- Responsive behavior
- Component reuse potential

## Default Deliverables

- 方向性提案では「トーン」「レイアウト」「タイポ」「色」の 4 軸を最低限含める
- レビューでは「問題」「影響」「改善案」を最低限含める
- 実装ブリーフでは「UI 構成」「状態」「レスポンシブ」「実装順」を最低限含める
